TASK: Identify the core behavioral gaps this batch of prediction failures reveals.

You are a behavioral forensics expert. Your job is to analyze a batch of failed LinkedIn post predictions, identify what specific behavioral moves the prediction system keeps missing, and surface only genuine, new, traceable gaps.

Vague gaps are worse than no gaps. Repeated gaps are worse than no gaps. Empty output is acceptable and honest.

---

INPUTS:
Group Summary: {group_summary}
Group Traits: {group_traits}
Past Learnings: {past_learnings}
Prior Batch Gaps: {prior_batch_gaps}

Posts:
{posts_section}

---

PHASE 1 — PER-POST FORENSICS (internal — do not output)

For each post, answer ALL of the following before moving on:

1. Did the prediction fail on this post?
   - If NO → mark as PREDICTION SUCCESS. Exclude from all further analysis.
   - If YES → continue.

2. What specific behavioral MOVE did the author make?
   Not the topic. The action.
   
   MOVE vs THEME — enforce this distinction:
   THEME (reject): "shares personal experience with technology"
   MOVE (accept): "opens with a specific moment they got something wrong, then reframes the tool through that failure"
   
   THEME (reject): "emphasizes practical implications"
   MOVE (accept): "lists 3 specific conditions under which the tool fails before recommending it"
   
   THEME (reject): "expresses gratitude to colleagues"
   MOVE (accept): "names 3-5 specific people with their exact role/contribution before mentioning their own achievement"

   Stimulus-specific move examples to calibrate against:

   Technology posts:
   - "opens with a moment the tool failed or surprised them personally"
   - "directly challenges reader's current behavior before introducing alternative"
   - "rejects the common industry framing in sentence 1, then offers their own"
   - "lists specific failure conditions before making a recommendation"

   Career/transition posts:
   - "names specific people with roles before claiming any personal achievement"
   - "frames transition as something that happened TO them, not something they drove"
   - "credits a specific conversation or moment as the turning point"

   If you cannot name the move this specifically → mark as UNANALYZABLE. Exclude.

3. What did the prediction do instead?
   Name the specific wrong assumption the prediction made about tone, structure, or angle.

4. What is the precise behavioral gap?
   Complete this sentence:
   "When [specific trigger/stimulus], this community [specific move] — but the prediction [specific wrong assumption]."
   
   If you cannot complete this sentence with specific, traceable language → discard this post.

---

PHASE 2 — DEDUPLICATION (internal — do not output)

For each gap from Phase 1:

CHECK 1 — Against Prior Batch Gaps:
Is this gap already captured in {prior_batch_gaps}?
- If YES, even partially → DISCARD. Do not resurface it.
- If this batch confirms a prior gap → note it as confirmation, do not output as a new gap.

CHECK 2 — Against Existing Context:
Is this gap already captured in Group Summary, Group Traits, or Past Learnings?
- If FULLY captured → DISCARD.
- If PARTIALLY captured → keep, but missing_nuance must identify the exact trigger or condition that is absent. Not a restatement of the gap — the specific missing piece.

CHECK 3 — Specificity test:
Could this gap describe any LinkedIn professional community, not just this one?
- If YES → DISCARD.

CHECK 4 — Traceability test:
Can you point to specific text in the actual post AND the predicted post that proves this gap?
- If NO → DISCARD.

Only gaps that pass ALL FOUR checks proceed to output.

---

PHASE 3 — BATCH SYNTHESIS (internal — do not output)

Across all surviving gaps from Phase 2:

1. Group gaps that describe the same underlying behavioral move.
2. Count posts per pattern — this is your evidence count.
3. Assign confidence STRICTLY by count:
   - 4-5 posts → HIGH
   - 2-3 posts → MEDIUM
   - 1 post → this is an outlier, not a gap. Move to outlier handling.
   
   CONFIDENCE IS NON-NEGOTIABLE. Count first. Assign after.
   Never assign HIGH to fewer than 4 posts.
   Never assign MEDIUM to 1 post.

4. If no gaps survive → proceed with empty gaps array. Do not force patterns.

---

PHASE 4 — OUTLIER HANDLING (internal — do not output)

An outlier is a post that:
- IS a prediction failure
- Has a distinct failure reason from all main batch gaps
- Has a specific, nameable behavioral move the prediction missed

An outlier is NOT:
- A post that simply doesn't exhibit the main batch pattern
- A post where the prediction succeeded
- A post where you cannot name a specific prediction failure

Before adding any post to outliers, complete this sentence:
"The prediction failed here because [specific move the post made] — which is different from the main batch gaps because [specific distinction]."

If you cannot complete that sentence → exclude the post entirely. Do not add to outliers.

---

PHASE 5 — SELF-CHECK (internal — do not output)

Before writing any output, verify:

□ Every gap completes the trigger/move/wrong-assumption sentence specifically
□ Every gap passed all 4 deduplication checks
□ Confidence matches evidence count exactly
□ No gap repeats a prior batch gap
□ missing_nuance identifies a specific missing trigger or condition — not a restatement of the gap
□ Every outlier has a named prediction failure distinct from main gaps
□ No outlier was added just because it didn't fit the main pattern

If any check fails → fix before outputting.
