TASK: Identify the core behavioral gaps this batch of prediction failures reveals.

You are a behavioral forensics expert analyzing failed LinkedIn post predictions.
Your job is to surface what this professional community genuinely believes and values
that the prediction system keeps missing.

Vague gaps are worse than no gaps.
Format and style observations are not behavioral gaps.
Stimulus-response patterns without an underlying belief are not behavioral gaps.
Empty output is acceptable and honest.

---

INPUTS:
Group Summary: {group_summary}
Group Traits: {group_traits}
Past Learnings: {past_learnings}
Prior Batch Gaps: {prior_batch_gaps}

Posts:
{posts_section}

Each post contains:
- Actual Post: what the person actually wrote
- Predicted Post: what the model generated

Both are required. If either is missing → mark UNANALYZABLE. Exclude entirely.

---

PHASE 1 — PER-POST FORENSICS (internal — do not output)

For each post:

STEP 1: Did the prediction fail?
- NO → PREDICTION SUCCESS. Exclude entirely. Do not analyze further.
- YES → continue.

STEP 2: What did the author actually DO?

Identify the specific move. Not the topic. Not the format. Not the length.
The move is what reveals who they are — what they believe or value professionally.

Enforce this distinction before proceeding:

REJECT these — they are not moves:
- "writes a short reply" → format
- "uses specific names" → style
- "shares personal experience" → theme
- "emphasizes practical implications" → generic
- "responds briefly to the post" → stimulus-response pattern

ACCEPT these — they reveal belief or value:
- "opens by naming what they got wrong before explaining what works"
- "credits a specific person's contribution before claiming personal achievement"
- "rejects the dominant industry framing in sentence 1, then offers a bounded alternative"
- "withholds the playbook publicly and routes to private follow-up"
- "anchors recommendation in a specific failure condition, not a general use case"

Test before accepting:
"Does this move reveal something about what this person believes
or values professionally — independent of the specific post topic?"
- YES → proceed to Step 3
- NO → format/style/theme observation. Discard. Do not proceed.

STEP 3: What did the prediction assume instead?
Name the specific wrong assumption about values, beliefs, or professional priorities.
Not about length, tone, emoji use, or format.

STEP 4: What is the gap?
Complete this sentence:
"[Specific move] — but the prediction assumed [specific wrong belief about this community]."

The move is the primary signal. The situation is background context only.
Do not lead with "When [stimulus]..." — lead with what the author did.

If you cannot complete this sentence with traceable, specific language → DISCARD.

---

PHASE 2 — DEDUPLICATION (internal — do not output)

For each gap from Phase 1, run all four checks. All must pass.

CHECK 1 — Prior Batch Gaps:
Is this gap already captured in {prior_batch_gaps}?
- YES, even partially → DISCARD.
- Confirmed again → note as prior_gap_confirmation, do not output as new gap.

CHECK 2 — Existing Context:
Is this gap already in Group Summary, Group Traits, or Past Learnings?
- FULLY captured → DISCARD.
- PARTIALLY captured → keep only if missing_nuance identifies the exact
  condition or dimension that is absent. Must be specific — not a restatement.

CHECK 3 — Community specificity:
Could this gap describe any professional LinkedIn community?
- YES → DISCARD. Must be specific to this community of Microsoft engineers
  and startup operators — not generic professional behavior.

CHECK 4 — Traceability:
Can you point to specific text in the actual post proving the move?
And specific text in the predicted post proving the assumption was wrong?
- NO → DISCARD.

---

PHASE 3 — BATCH SYNTHESIS (internal — do not output)

Across all gaps that passed Phase 2:

1. Group gaps that describe the same move appearing across multiple posts.
   Same move = same action, same professional value revealed,
   even if surface topics differ across posts.

2. Count posts per group → this is your evidence count.
   Do not estimate. Count exactly.

3. Assign confidence STRICTLY by count. No exceptions:
   - 4-5 posts → HIGH
   - 2-3 posts → MEDIUM
   - 1 post → outlier only. Never a gap.

4. If no gaps survive → empty gaps array. Do not force patterns.

---

PHASE 4 — OUTLIER HANDLING (internal — do not output)

An outlier is a prediction failure where:
- The move is specific and nameable
- It is distinct from all main batch gaps
- It is not already in existing context
- It reveals something about this community's values or beliefs

Before adding any outlier, complete:
"The prediction missed [specific move] —
which is distinct from main batch gaps because [specific distinction]."

Cannot complete it → exclude.
Format or style observation → exclude.
Already in existing context → exclude.
Doesn't reveal a belief or value → exclude.

---

PHASE 5 — SELF-CHECK (internal — do not output)

□ Every behavioral_gap leads with the move, not the stimulus
□ No gap is a format, length, or style observation
□ No gap is a stimulus-response pattern without an underlying move
□ Every gap passed all 4 deduplication checks
□ Confidence matches exact evidence count — no exceptions
□ No gap repeats a prior batch gap
□ missing_nuance names a specific missing condition — not a restatement
□ Every outlier names a distinct move not in main gaps or existing context
□ No outlier added just because it didn't fit the main pattern
□ No gap could describe any generic LinkedIn professional community

If any check fails → fix before outputting.

---

OUTPUT FORMAT:
Return only valid JSON. No markdown. No commentary.

{

  "gaps": [
    {

    }
